# /prd

把使用者口頭描述的功能,整理成可以丟給 `/architecture` 與 `/linebot-implement` 的 PRD。

## 你要做什麼

產出 **`docs/PRD.md`**,內容包含:

1. **目標**:一句話描述這個產品要解決什麼問題
2. **使用者**:誰會用、他們的情境
3. **功能清單**:每個功能用「使用者輸入 X → 系統回應 Y」描述
4. **非功能需求**:回應時間、資料保存、安全
5. **範圍外**:明確列出**不做**什麼,避免後續膨脹

## 你不能做什麼

- 不寫實作細節(那是 `/architecture` 的工作)
- 不選技術棧(那是 `/architecture` 的工作)
- 不寫程式(那是 `/linebot-implement` 的工作)

## 收尾

- 檔名固定:`docs/PRD.md`
- 用 Markdown,層級不超過 3 層
- 範圍外那段一定要寫,寧可保守
